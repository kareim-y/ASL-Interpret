# ASL Letter Recognition — ENSF 511 Final Project

Real-time American Sign Language (ASL) letter classification using transfer learning on MobileNetV2, exported to TFLite for edge deployment. The repo contains three main components: a data collection UI, a training notebook, and a live inference UI.

---

## Repository Structure

```
asl-recognition/
├── asl_data_collect_ui.py      # Webcam-based data collection tool
├── asl_training_workflow.ipynb # End-to-end model training notebook
├── asl_live_ui.py              # Real-time ASL inference UI
├── artifacts/                  # Generated by training — models & metadata
│   ├── metadata.json
│   ├── asl_fp16.tflite
│   ├── asl_int8.tflite
│   └── asl_none.tflite
└── data/                       # Dataset directory (A–Z class folders)
    ├── A/
    ├── B/
    └── ...
```

## Components

### 1. Data Collection UI — `asl_data_collect_ui.py`

A webcam utility for capturing and labelling your own ASL images. Useful for augmenting public datasets or building a custom dataset from scratch.

**Usage:**

```bash
python asl_data_collect_ui.py --output-dir data --letter A --num-images 200
```

**Controls (while running):**
| Key | Action |
|-----|--------|
| `Space` | Capture a frame |
| `n` | Move to next letter |
| `q` | Quit |

---

### 2. Training Notebook — `asl_training_workflow.ipynb`

An end-to-end Jupyter notebook optimised for Apple Silicon (MPS via `tensorflow-metal`). It covers:

- Dataset loading from `A`–`Z` class folders with configurable train/val/test splits
- MobileNetV2 backbone with a custom classification head
- Two-stage training: head-only first, then partial fine-tuning of the backbone
- Optional magnitude-based weight pruning (`tensorflow-model-optimization`)
- TFLite export in three quantization modes: `none`, `fp16`, and `int8`
- A final JSON summary report written to `artifacts/`

> **Note:** `J` and `Z` are motion-based signs and will typically score lower with a static image classifier.

**Dataset sources used in development:**

- Collected using the `data_collect.py` script
- [Kaggle ASL Dataset](https://www.kaggle.com/datasets/ayuraj/asl-dataset)
- [Mendeley ASL Data](https://data.mendeley.com/datasets)

**Key hyperparameters (configured at the top of the notebook):**

| Parameter         | Default | Description                |
| ----------------- | ------- | -------------------------- |
| `IMAGE_SIZE`      | 224     | Input resolution (px)      |
| `BATCH_SIZE`      | 32      | Training batch size        |
| `VAL_SPLIT`       | 0.15    | Validation fraction        |
| `TEST_SPLIT`      | 0.10    | Test fraction              |
| `LR_HEAD`         | 1e-3    | Learning rate (head phase) |
| `ENABLE_PRUNING`  | `False` | Enable weight pruning      |
| `TARGET_SPARSITY` | 0.50    | Pruning target sparsity    |

---

### 3. Live Inference UI — `asl_live_ui.py`

A real-time OpenCV window that reads from your webcam, detects your hand with MediaPipe, crops the region of interest, and runs inference with a TFLite model. Predicted letters are accumulated into a running word buffer.

**Usage:**

```bash
python asl_live_ui.py --model-dir artifacts --model-name asl_fp16.tflite
```

**All CLI arguments:**

| Flag              | Default           | Description                                                  |
| ----------------- | ----------------- | ------------------------------------------------------------ |
| `--model-dir`     | `artifacts`       | Directory containing `metadata.json` and the `.tflite` model |
| `--model-name`    | `asl_fp16.tflite` | TFLite model filename                                        |
| `--camera-id`     | `0`               | Webcam device index                                          |
| `--padding`       | `30`              | Pixel padding around the detected hand bounding box          |
| `--stable-frames` | `8`               | Consecutive frames required before committing a letter       |
| `--conf-thresh`   | `0.70`            | Minimum confidence threshold for a prediction to count       |

**Runtime controls:**

| Key                    | Action                            |
| ---------------------- | --------------------------------- |
| `q`                    | Quit                              |
| `c`                    | Clear the word buffer             |
| `Backspace` / `Delete` | Remove the last committed letter  |
| `Space`                | Insert a space in the word buffer |

**Fallback behaviour:** if MediaPipe Hands is unavailable, the UI falls back to a centre-crop of 60% of the frame.

## Installation

### Requirements

- Python 3.9+
- Apple Silicon recommended (MPS acceleration via `tensorflow-metal`); CPU-only works on any platform

### Setup

```bash
# 1. Create a conda environment
conda create -n asl_train python=3.10 -y
conda activate asl_train

# 2. Install dependencies
pip install "numpy<2" tensorflow==2.15.1 tensorflow-metal==1.1.0 tf_keras==2.15.1 tensorflow-model-optimization
pip install opencv-python mediapipe matplotlib scikit-learn jupyter ipykernel
python -m ipykernel install --user --name asl_train_gpu2 --display-name "Python (asl_train_gpu2)"
```

> On non-Apple platforms, omit `tensorflow-metal`. Standard `tensorflow` will use CPU or CUDA automatically.

## Workflow

```
Collect images                     Train model                       Run live UI
─────────────────              ─────────────────────            ─────────────────────────
asl_data_collect_ui.py  →   asl_training_workflow.ipynb    →        asl_live_ui.py
```

1. **Collect** (optional): use `asl_data_collect_ui.py` to capture custom training images, or download one of the public datasets linked above.
2. **Train**: open and run `asl_training_workflow.ipynb` top-to-bottom. Trained models and metadata are saved to `artifacts/`.
3. **Infer**: run `asl_live_ui.py`, pointing it at the `artifacts/` directory. The `fp16` model is recommended as a balance between size and accuracy. Use `int8` for maximum speed on low-power hardware.

## Model Artifacts

After training, the `artifacts/` directory will contain:

| File                   | Description                                     |
| ---------------------- | ----------------------------------------------- |
| `metadata.json`        | Class names and input image size                |
| `best_head.weights.h5` | Weights from the head-only training phase       |
| `best_finetuned.keras` | Full fine-tuned Keras model                     |
| `best_float.keras`     | Float model evaluated on the test set           |
| `asl_none.tflite`      | TFLite — no quantization                        |
| `asl_fp16.tflite`      | TFLite — FP16 quantization (recommended)        |
| `asl_int8.tflite`      | TFLite — INT8 quantization (smallest / fastest) |
| `summary.json`         | Full training and evaluation report             |

## Notes & Limitations

- **Static signs only.** `J` and `Z` require motion to sign correctly; single-frame classification accuracy on these letters will be lower than the rest of the alphabet.
- **Lighting sensitivity.** Model performance degrades in poor or uneven lighting. Collect training data under varied conditions for best results.
- **One hand only.** The live UI tracks a single hand (`max_num_hands=1`).
- **Stable-frame logic.** A letter is only committed after it is predicted consistently for `--stable-frames` consecutive frames above `--conf-thresh`. Tune these values to balance responsiveness against false positives.
